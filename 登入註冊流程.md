# 登入與註冊流程

本文說明目前系統的帳號註冊、登入、session、個人資料與密碼修改流程。

## 1. 相關資料表

後端啟動時會呼叫 `ensureDatabaseExists`、`ensureAuthTables` 與 `ensureCvTables`，自動建立資料庫與資料表。預設資料庫名稱為 `hrai`，可由 `HRAI_DATABASE` 或 `MYSQL_DATABASE` 覆蓋。

帳號相關表：

| 表名 | 用途 |
| --- | --- |
| `users` | 保存 email、使用者名稱、頭像設定、密碼 hash 與 salt。 |
| `auth_tokens` | 保存登入 token 的 hash、到期時間與對應 user。 |

`auth_tokens.user_id` 以外鍵連到 `users.id`，使用者刪除時 token 會一併刪除。

## 2. 密碼保存方式

密碼不保存明文。

註冊與修改密碼時：

1. 後端產生 16 bytes 隨機 salt。
2. 使用 Node `crypto.pbkdf2(password, salt, 100000, 64, 'sha512')` 計算 hash。
3. 將 hash 寫入 `users.password_hash`，salt 寫入 `users.password_salt`。

登入時：

1. 用 email 讀取使用者。
2. 用資料庫中的 salt 對輸入密碼重新計算 hash。
3. 比對 hash 是否一致。

## 3. 註冊流程

前端入口：登入頁的 `註冊` 分頁。

### 3.1 請求驗證碼

API：`POST /api/auth/request-code`

請求內容：

```json
{
  "email": "name@company.com"
}
```

後端行為：

- 產生 6 位數驗證碼。
- 存入記憶體 `verificationCodes` Map。
- 有效期 60 秒。
- 目前驗證碼只輸出在後端 console log，前端提示「請查看後端日誌」。

### 3.2 建立帳號

API：`POST /api/auth/register`

請求內容：

```json
{
  "email": "name@company.com",
  "password": "secret123",
  "code": "123456"
}
```

後端行為：

- 檢查 email、password、code 是否存在。
- 校驗驗證碼是否存在、未過期且相符。
- 檢查 email 是否已註冊；已存在則回 `409`。
- 寫入 `users`。
- 初始 `username` 取 email 的 `@` 前段。
- 初始 `avatar_text` 取 username 第一個字元。
- 初始 `avatar_bg_color` 為 `#334155`。
- 註冊成功後刪除該 email 的驗證碼。

前端成功後會切回登入分頁，並把註冊 email 帶入登入欄位。

## 4. 登入流程

API：`POST /api/auth/login`

請求內容：

```json
{
  "email": "name@company.com",
  "password": "secret123"
}
```

後端行為：

- 檢查 email 與 password。
- 讀取使用者並校驗密碼。
- 產生 32 bytes 隨機 token。
- 以 SHA-256 保存 token hash 到 `auth_tokens.token_hash`。
- 寫入 `expires_at`。
- 登入時會先刪除該使用者舊 token 及已過期 token。

回應內容：

```json
{
  "token": "...",
  "expiresAt": "2026-05-11T12:00:00.000Z",
  "user": {
    "id": 1,
    "mail": "name@company.com",
    "username": "name",
    "role": "normal",
    "avatarText": "N",
    "avatarBgColor": "#334155"
  }
}
```

前端會保存：

- `innerai_auth`：`{ token, expiresAt }`
- `innerai_user`：使用者資料
- 若勾選「記住我」，另存 `innerai_remember_email`

## 5. Session 與路由保護

前端路由只有兩個主要路徑：

- `/`：登入/註冊頁
- `/main`：登入後工作台

`src/router/index.js` 會檢查 localStorage 中的 `innerai_auth`：

- 沒有 token：導回 `/`
- `expiresAt` 無效或已過期：清除 localStorage 並導回 `/`
- token 未過期：允許進入 `/main`

`Main.vue` 也會每秒檢查一次 token 到期時間；到期後清除登入資料並返回登入頁。

後端需要授權的 API 使用 `Authorization: Bearer <token>`，由 `getAuthedUser` 查詢 `auth_tokens` 並確認 `expires_at > NOW()`。

## 6. 個人資料

### 6.1 讀取個人資料

API：`GET /api/auth/profile`

要求：

```http
Authorization: Bearer <token>
```

回傳目前使用者資料。

### 6.2 更新個人資料

API：`POST /api/auth/profile`

請求內容：

```json
{
  "username": "HR User",
  "avatarText": "HR",
  "avatarBgColor": "#334155"
}
```

校驗規則：

- `username` 必填且不超過 80 字。
- `avatarText` 必填，最多保留 6 個字元。
- `avatarBgColor` 若提供，必須是 `#RRGGBB` 格式。

成功後前端會更新 `innerai_user`。

## 7. 修改密碼

API：`POST /api/auth/change-password`

請求內容：

```json
{
  "currentPassword": "old-password",
  "newPassword": "new-password"
}
```

後端行為：

- 驗證登入 token。
- 檢查目前密碼是否正確。
- 新密碼長度至少 6 個字元。
- 產生新的 salt 與 hash。
- 更新 `users.password_hash`、`users.password_salt`。

## 8. Token 有效期設定

預設 token 有效期為 1 小時。

可用環境變數覆蓋：

```env
AUTH_AUTO_LOGIN_TTL_MS=3600000
AUTH_AUTO_LOGIN_TTL_MINUTES=60
```

若兩者都未設定，使用預設 60 分鐘。

## 9. 目前限制

- 註冊驗證碼未串接 Email/SMS 發送服務，只輸出於後端日誌。
- 後端重啟會清空尚未使用的驗證碼，因為驗證碼存在記憶體 Map。
- 前端路由只根據 localStorage 的到期時間做快速保護；真正授權仍由後端 token 表校驗。
