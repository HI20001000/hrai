# 產品說明：HRAI / InnerAI HR 系統

本文整理目前系統的主要功能、使用流程與模組邊界，供使用者、產品維護者與開發者快速理解現況。

## 1. 系統定位

HRAI 是招聘與人員安排工作台，目標是把「職缺建立、CV 解析、候選人投遞、AI 匹配、候選人跟進、內部項目人員安排」放在同一套系統內處理。

典型使用者：

- HR：建立職缺、上傳 CV、查看 AI 分析結果、更新候選人狀態。
- 招聘負責人：維護職位字典、檢視候選人與職缺匹配。
- 項目/交付管理者：維護內部人員與項目人員安排。
- 系統管理者：維護帳號設定、職位字典與基礎資料。

## 2. 導航與模組

登入後主工作台包含以下模組：

| 模組 | 主要用途 |
| --- | --- |
| 職缺管理 | 建立職缺、更新職缺狀態、查看該職缺下的候選人清單。 |
| 候選人管理 | 集中查看所有職缺的候選人投遞、AI 分析檔、CV、狀態與備註。 |
| 上傳候選人 CV | 選擇開放中的職缺，單份或批量上傳 CV，解析後確認並匹配。 |
| 名單管理 | 維護內部人員與候選人 Blacklist。 |
| 項目管理 | 維護項目、項目人員安排、調動與 CSV 匯入。 |
| 設定 | 編輯個人資料、修改密碼、維護職位字典。 |

## 3. 招聘主流程

### 3.1 維護職位字典

入口：`設定` 或 `職缺管理` 頁面的職位字典面板。

職位字典資料保存於 `finance-job-positions.json`。每個職位包含：

- 職位描述、行業背景、職位關鍵字
- 核心職責、必備技能、加分技能、證照
- 專案經驗、工作年資、候選人偏好
- 薪資範圍與匹配權重

更新職位字典後，只影響後續新建職缺或重新觸發的匹配；既有職缺會保留建立當下的職位快照。

### 3.2 建立與管理職缺

入口：`職缺管理`。

可操作內容：

- 從職位字典建立職缺。
- 設定職缺狀態：`開放中`、`草稿`、`已關閉`。
- 查看職缺快照中的職位要求。
- 查看目前職缺候選人清單。
- 刪除職缺；刪除時會一併清理該職缺下的投遞與相關 CV 檔案。

職缺建立時，系統會將職位字典固化成 `job_snapshot_json`，確保後續字典調整不會改動已建立職缺的判斷基準。

### 3.3 上傳與解析 CV

入口：`上傳候選人 CV`。

流程：

1. 從開放中的職缺列表選擇目標職缺。
2. 選擇一份或多份 CV。
3. 系統先快取檔案，再呼叫 LLM 解析。
4. 使用者確認 AI 分析結果，可人工編輯欄位。
5. 確認後寫入候選人、CV、投遞與匹配資料。

支援的 AI 分析欄位：

- 姓名、Email、電話
- 學歷、工作年限
- 語言、技術語言/工具、技術證書
- 所屬行業、項目經歷
- 目標職位、期望薪資、入職意願

工作年限若 LLM 未抽取成功，後端會在 CV 原文有明確工作日期時做確定性補算。

### 3.4 候選人管理與跟進

入口：`候選人管理`。

可操作內容：

- 查看所有職缺下的候選人投遞。
- 搜尋職位、候選人、狀態、期望職位、匹配職位、電話、備註與檔案。
- 更新候選人狀態與備註。
- 預覽/下載 CV 檔案。
- 預覽 AI 分析檔，並直接修正抽取欄位。
- 將候選人加入 Blacklist。
- 將候選人加入項目人員名單。
- 批量刪除、批量加入/取消 Blacklist、批量上傳 CV。

候選人狀態包含：

- 簡歷篩選中
- 簡歷篩選 - HR通過 / HR不通過
- 簡歷篩選 - 部門通過 / 部門不通過
- 簡歷篩選不通過
- HR面試 / HR面試不通過
- 部門面試 / 部門面試不通過
- 薪資評估、發出Offer、已入職、轉職位
- 面試不出現/聯繫不上、拒絕Offer、HR撤銷入職

### 3.5 AI 匹配

系統包含三類 LLM prompt：

- CV 特徵抽取：`server/prompts/cv-feature-extraction.prompt.txt`
- 職位初篩：`server/prompts/job-shortlist.prompt.txt`
- 職位精排與單職缺匹配：`job-rerank.prompt.txt`、`job-single-match.prompt.txt`

匹配會輸出分數、等級、匹配職位、優勢與差距，用於候選人列表與 AI 分析檔預覽。

## 4. 名單管理

入口：`名單管理`。

包含兩個分頁：

### 4.1 內部人員

維護欄位：

- 姓名、部門、組別、職稱
- Email、電話、直屬主管
- 狀態：啟用中 / 停用
- 備註

內部人員可作為項目負責人與項目人員安排的來源。

### 4.2 候選人 Blacklist

維護欄位：

- 顯示名稱、電話、Email
- 原因、狀態、備註

系統會以標準化電話與 Email 進行命中判斷；候選人清單命中 Blacklist 時會顯示風險提示。

## 5. 項目管理

入口：`項目管理`。

可操作內容：

- 新增/編輯/刪除項目。
- 項目狀態：籌備中、進行中、暫停、已完成。
- 指定項目負責人、起止日期與備註。
- 從內部人員加入項目。
- 從已入職候選人轉入項目人員。
- 更新項目人員狀態：在組、已調動、已移出。
- 調動人員到其他項目。
- CSV 匯入項目人員。
- 查看人員名單預覽與調動來源。

項目調動會寫入 movement 記錄，便於追蹤人員來源、去向與操作時間。

## 6. 帳號與設定

入口：`設定`。

可操作內容：

- 讀取與更新個人資料：使用者名稱、頭像文字、頭像背景色。
- 修改密碼。
- 維護職位字典。

登入狀態由前端 localStorage 中的 `innerai_auth` 和 `innerai_user` 保存；前端會定時檢查 token 到期時間，過期後導回登入頁。

## 7. 使用建議

- 建議先維護好職位字典，再建立職缺，避免職缺快照不完整。
- 上傳 CV 後先確認 AI 分析欄位，尤其是工作年限、項目經歷與目標職位。
- 候選人備註可用於記錄跟進原因；長備註可透過滑鼠懸停查看全文。
- 對於不應再次聯絡或有資料風險的候選人，應加入 Blacklist，便於後續上傳時提示。
- 已入職候選人可轉入項目管理，形成從招聘到項目安排的閉環。
